- **목표:** 전체 코스를 마친 후 커리어에 딥러닝 역량을 자신 있게 추가하고, AI 기반 사회를 만들어가는 데 기여하는 것을 목표로 합니다.

# < Intro to Deep Learning >
## 1. 신경망(Neural Network)이란 무엇인가?

### 1). 단일 뉴런(Single Neuron)의 개념과 집 가격 예측 예시

- **선형 회귀의 한계:** 집 크기($x$)로 집 가격($y$)을 예측할 때, 직선을 그리면 특정 크기 이하에서는 가격이 음수(Negative)가 되는 문제가 발생합니다.
    
- **ReLU 함수 적용:** 집 가격은 0보다 작을 수 없으므로, 일정 구간까지는 0을 유지하다가 그 이후 직선으로 증가하는 함수를 적용합니다.
    
- **ReLU (Rectified Linear Unit):**
    
    - $0$과 입력값 중 큰 값을 취하는 함수 ($\max(0, x)$).
    - 딥러닝에서 가장 흔하게 쓰이는 기본 활성화 함수입니다.
        
- **단일 뉴런 모델:** 집 크기($x$)를 입력받아 ReLU 함수를 거쳐 예측 가격($y$)을 출력하는 작은 원 하나가 **가장 단순한 형태의 인공 뉴런**입니다.
    

### 2). 신경망의 확장: Lego 블록 쌓기

단일 뉴런을 **레고 블록**처럼 여러 개 결합하고 쌓아 올리면 커다란 신경망이 완성됩니다.

#### 예시: 4가지 특성(Feature)을 활용한 집 가격 예측

- **입력 특성 ($x$):**
    
    1. 집 크기 (Size)
    2. 침실 수 (Bedrooms)
    3. 우편번호 (Zip / Postal Code)
    4. 동네 소득 수준 (Wealth)
        
#### 사람이 생각하는 중간 개념 (특성 조합):

- **가족 규모 (Family Size):** 집 크기 + 침실 수
- **보행 편의성 (Walkability):** 우편번호
- **학교 수준 (School Quality):** 우편번호 + 동네 소득 수준
- **최종 가격 ($y$):** 가족 규모 + 보행 편의성 + 학교 수준을 종합하여 결정.
    

### 3). 인공지능의 마법: 완전 연결층 (Dense Layer)

- **자동 특성 추출 (Automated Feature Extraction):**
    
    - 실무에서는 사람이 일일이 "첫 번째 노드는 가족 규모, 두 번째 노드는 보행 편의성"이라고 지정해 주지 않습니다.
        
    - 입력 데이터 $X$와 정답 $Y$만 주면, 신경망이 **중간 노드(은닉층, Hidden Layer)의 역할을 스스로 학습**하여 최적의 의미를 찾아냅니다.
        
- **조밀하게 연결된 레이어 (Densely Connected Layer):**
    
    - 은닉층의 모든 노드는 입력층의 **모든 특성 4가지($x_1 \sim x_4$)를 전부 전달받아** 스스로 필요한 정보를 계산합니다.
        
- **학습 능력:** $X$와 $Y$ 데이터가 충분히 제공되면, 신경망은 $X$에서 $Y$로 매핑하는 정교한 함수를 매우 뛰어난 성능으로 찾아냅니다.

## 2. 신경망과 지도 학습

### 1). 지도 학습(Supervised Learning)의 정의

- **개념:** 입력값($x$)을 받아 출력값($y$)으로 매핑하는 함수를 학습하는 머신러닝 방식입니다.
    
- **경제적 가치:** 현재 인공지능이 창출하는 거의 모든 경제적 가치는 지도 학습 기반의 신경망에서 나옵니다.
    
- **핵심 과제:** 특정 문제 해결을 위해 어떤 데이터를 $x$(입력)로 두고 $y$(출력)로 설정할지 적절히 정의하는 것입니다.
    

### 2). 주요 응용 분야 및 입출력($x \to y$) 예시

|**응용 분야**|**입력 (x)**|**출력 (y)**|**비고**|
|---|---|---|---|
|**부동산 (Real Estate)**|주택의 특성 (크기, 침실 수 등)|주택 가격|기본 구조 사용|
|**온라인 광고 (Online Ads)**|광고 정보 + 사용자 정보|클릭 여부 (Click / No Click)|**현재 딥러닝 분야 중 가장 수익성이 높은 응용 분야**|
|**컴퓨터 비전 (Computer Vision)**|이미지|이미지 범주/태그 (예: 1~1000 번호)|사진 태깅 등에 활용|
|**음성 인식 (Speech Recognition)**|오디오 클립|텍스트 대화록 (Text Transcript)|최근 큰 진전|
|**기계 번역 (Machine Translation)**|특정 언어 문장 (예: 영어)|다른 언어 문장 (예: 중국어)|-|
|**자율주행 (Autonomous Driving)**|전방 카메라 이미지 + 라다(Radar) 정보|주변 차량의 위치|복합 시스템에 탑재|

### 3). 데이터 유형별 신경망 아키텍처

1. **표준 신경망 (Standard Neural Network):**
    
    - 주택 가격 예측, 온라인 광고 추천 등에 주로 활용됩니다.
        
2. **합성곱 신경망 (CNN, Convolutional Neural Network):**
    
    - **이미지(Computer Vision)** 데이터 처리에 특화되어 있습니다.
        
3. **순환 신경망 (RNN, Recurrent Neural Network):**
    
    - **시퀀스(Sequence) 데이터** 처리에 최적화되어 있습니다.
        
    - **오디오(Audio):** 시간에 따른 1차원 시계열/시간적 시퀀스 데이터.
        
    - **언어(Text):** 단어나 알파벳이 순차적으로 등장하는 시퀀스 데이터.
        
4. **하이브리드 / 맞춤형 신경망 (Custom Hybrid Architecture):**
    
    - 자율주행처럼 이미지(CNN)와 라다 정보 등 서로 다른 유형의 입력을 동시에 처리해야 하는 복잡한 시스템에 적용됩니다.
        

### 4). 구조화 데이터 vs 정형/비정형 데이터 (Structured vs Unstructured Data)

#### **구조화 데이터 (Structured Data)**

- **특징:** 데이터베이스 형태로 정리되어 있으며, 각 피처(기능/특성)의 의미가 명확합니다.
    
- **예시:** 집 크기, 침실 수, 사용자의 연령, 광고 클릭 데이터 등.
    
- **비즈니스 가치:** 기업들이 보유한 거대 데이터베이스를 바탕으로 광고, 제품 추천, 정확한 예측을 수행하여 **단기적으로 매우 큰 경제적 가치**를 창출합니다.
    

#### **비정형 데이터 (Unstructured Data)**

- **특징:** 정해진 규격이 없는 데이터입니다.
    
- **예시:** 원시 오디오(Audio), 이미지(Image), 텍스트(Text) 등 (피처: 픽셀 값, 단어 자체).
    
- **발전 양상:** 인간은 비정형 데이터를 직관적으로 잘 이해하지만, 과거 컴퓨터에게는 매우 어려운 과제였습니다. 최근 딥러닝의 등장으로 컴퓨터의 비정형 데이터 처리 능력이 획기적으로 향상되었습니다.
    
- **미디어 조명:** "신경망이 고양이를 인식했다"처럼 대중이 직관적으로 이해하기 쉽고 흥미롭기 때문에 미디어에 자주 소개됩니다.

## 3. AI/딥러닝 연구자 및 입문자를 위한 조언

1. **독서량보다 직관을 중시할 것 (Contrarian Approach):**
    
    - 논문을 너무 많이 읽지 말고, 조금만 읽은 후 남들이 남발하거나 당연하게 생각하는 것 중 "잘못되었다고 느낀 부분"을 찾아 도전할 것.
        
    - 다른 사람들이 "말도 안 되는 소리"라고 비난할 때, 본인의 직관이 좋다는 전제하에 그 아이디어가 진정한 대혁신의 신호일 수 있음 (변분법 발표 초기 비난받았던 경험 사례).
        
2. **자신의 직관(Intuition)을 믿을 것:**
    
    - 직관이 옳다면 성공할 것이고, 틀리다면 무엇을 해도 안 될 테니 자신의 직관을 밀고 나가는 것이 가장 현명함.
        
3. **직접 코딩(Programming)을 절대 그만두지 말 것:**
    
    - 연구 성공의 열쇠는 미세한 구현 상의 결정에 달려 있으므로, 지속적인 프로그래밍 경험이 필수적임.
        
4. **지도교수(Advisor) 선택:**
    
    - 본인과 비슷한 신념과 가치관을 가진 지도교수를 만나는 것이 연구에 있어 전폭적인 지원과 유용한 조언을 얻는 길임.
        
5. **대학 PhD vs 기업 연구소:**
    
    - 현 컴퓨터공학과들은 프로그래밍(Programming) 중심에서 데이터를 보여주고 학습시키는 **"보여주기(Showing)"** 중심으로 전환되는 거대한 변화를 충분히 수용하지 못하고 있음.
        
    - 대학의 수용 능력(Bandwidth) 부족으로 인해 당분간은 기업(예: Google Brain Residency 등)에서의 교육 역할이 매우 중요할 것임.
