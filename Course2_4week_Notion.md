# < Decision Trees >
## 1. 의사결정나무(Decision Tree)란?

- **실무 활용도 높음:** 학계보다 실무 및 머신러닝 경진대회(Kaggle 등)에서 매우 자주 쓰이고 강력한 성능을 발휘하는 모델입니다.
    
- **기본 작동 방식:** 데이터의 특징(Feature)에 대해 단계별로 질문을 던지며 가지를 치고, 최종 판단을 내리는 스무고개 방식의 모델입니다.
    

## 2. 의사결정나무의 구조와 용어

강의에서는 **고양이 분류 예시**(_귀 모양_, _얼굴 모양_, _수염 유무_ $\rightarrow$ _고양이 여부 판단_)를 통해 구조를 설명합니다.

```
                 [ 루트 노드 ]            <-- 첫 번째 질문 (최상단)
                   /      \
      [ 의사결정 노드 ]   [ 의사결정 노드 ]  <-- 중간 질문 (특징 조건 검사)
        /        \         /        \
    [ 리프 ]   [ 리프 ]  [ 리프 ]   [ 리프 ]  <-- 최종 예측 결과 (최하단)
```

|**구성 요소**|**설명**|
|---|---|
|**루트 노드 (Root Node)**|트리의 맨 위에 위치하며, 데이터 평가가 시작되는 첫 번째 노드입니다.|
|**의사결정 노드 (Decision Node)**|특정 특징(Feature)의 값을 검사하여 데이터 방향(왼쪽/오른쪽)을 결정하는 중간 노드입니다.|
|**리프 노드 (Leaf Node)**|트리의 맨 아래쪽에 위치하며, 최종 분류 결과(예: _고양이_ vs _고양이 아님_)를 출력합니다.|

## 3. 데이터 및 특징(Feature) 종류

- **범주형 특징 (Categorical Features):** `귀 모양 = 뾰족함 / 접힘`, `수염 = 있음 / 없음` 처럼 몇 개의 명확한 범주(값)로 나뉘는 데이터입니다.
    
- **이진 분류 (Binary Classification):** 타깃값($Y$)이 0 또는 1(고양이인가? 아닌가?)의 두 가지로 나뉩니다.
    

## 4. 의사결정나무 학습의 목표

- 같은 데이터셋이라도 여러 형태의 트리를 만들 수 있습니다.
    
- 알고리즘의 목표는 **훈련 데이터(Training Set)를 잘 맞추면서도, 새로운 데이터(Cross-validation/Test Set)에 일반화(Generalization)가 잘 되는 최적의 트리를 찾는 것**입니다.

## 5. 의사결정나무 구축 절차 (Building Process)

10개의 동물 데이터(고양이 vs 개)를 분류하는 기본 알고리즘 절차입니다.

- **5.1. 루트 노드(Root Node) 분할:** 최상단 노드에서 기준이 될 특징(Feature)을 선택해 데이터를 두 그룹으로 나눕니다. (예: 귀 모양)
    
- **5.2. 가지(Branch) 분할 및 이동:** 나누어진 하위 데이터셋에 대해 또 다른 특징(예: 얼굴 모양, 수염 유무)을 기준 삼아 재귀적으로 분할을 반복합니다.
    
- **5.3. 리프 노드(Leaf Node) 생성:** 데이터 분할 기준이 충족되면 최종 예측값(고양이 또는 개)을 출력하는 리프 노드를 만듭니다.
    

## 6. 핵심 결정 요인 2가지 (Key Decisions)

의사결정나무 알고리즘을 구현할 때 반드시 결정해야 하는 두 가지 핵심 질문입니다.

### 1). 어떤 특징(Feature)을 기준으로 분할할 것인가?

- **순도(Purity) 극대화:** 한쪽 가지에 고양이만 모이거나 개만 모이도록 순도가 높아지는 방향(불순도가 낮아지는 방향)으로 분할 기준을 선택합니다.
    
- 다음 단계에서 배울 **엔트로피(Entropy)** 개념을 통해 이 불순도를 정량적으로 측정하고 최선의 분할 특징을 선택합니다.
    

### 2). 언제 분할을 멈출 것인가? (Stopping Criteria)

다음 조건 중 하나 이상을 만족할 때 분할을 멈추고 리프 노드를 생성합니다.

- **완전한 순도 달성:** 노드가 100% 고양이 또는 100% 개로만 이루어졌을 때.
    
- **최대 깊이(Maximum Depth) 도달:** 트리의 깊이가 미리 지정한 제한값에 도달했을 때. (_트리가 너무 깊어지는 것을 막아 **오버피팅(Overfitting)** 방지_)
    
- **순도 개선 미흡:** 분할을 통해 얻는 순도의 이득(Information Gain)이 일정 기준(Threshold)보다 낮을 때.
    
- **최소 샘플 수 미달:** 노드에 남아 있는 데이터 수가 일정 개수 이하일 때.
    

## 7. 요약 및 특징

- 의사결정나무는 여러 연구자들이 제안한 분할 방식과 정지 조건들이 결합되어 **구조가 다소 복잡해 보일 수 있지만, 실무에서 매우 강력하고 효과적**으로 작동합니다.
    
- 직접 모든 정지 규칙을 구현하기보다 오픈소스 패키지(예: Scikit-learn)를 활용하면 손쉽게 최적의 모델을 구축할 수 있습니다.

# < Decision Tree Learning >

## 1. Measuring Purity
- Entropy, 불순도 측정 지표
### 1). 엔트로피(Entropy)의 개요

#### (1). 정의 및 목적

- **불순도(Impurity) 측정 지표:** 엔트로피는 데이터 집합이 얼마나 섞여 있는지(불순한지)를 정량적으로 측정하는 함수입니다.
    
- **순도(Purity)와의 관계:** 데이터가 한 종류로만 구성되어 있으면 순도가 높고(불순도/엔트로피 = 0), 절반씩 섞여 있으면 순도가 가장 낮습니다(불순도/엔트로피 = 1).
    

#### (2). 양성 비율($p_1$)에 따른 엔트로피 변화 예시

- $p_1$은 전체 데이터 중 양성 클래스(예: 고양이, Label = 1)가 차지하는 비율입니다.
    
- **$p_1 = 0.5$ (50% : 50% 섞임):** 가장 불순한 상태이며, 엔트로피 $H(p_1) = 1$입니다.
    
- **$p_1 = 0.83$ 또는 $0.33$ (한쪽이 우세):** 치우친 상태로, 엔트로피 $H(p_1) \approx 0.65$ 또는 $0.92$로 감소합니다.
    
- **$p_1 = 1$ 또는 $0$ (100% 한 클래스만 존재):** 완전히 순수한 상태이며, 엔트로피 $H(p_1) = 0$입니다.
    
---
### 2). 엔트로피 수식 및 특징

#### (1). 계산 공식

- $p_1$을 고양이 비율, $p_0 = 1 - p_1$을 고양이가 아닌 비율이라 할 때 엔트로피 수식은 다음과 같습니다:
    
    $$H(p_1) = -p_1 \log_2(p_1) - p_0 \log_2(p_0)$$
    
    $$H(p_1) = -p_1 \log_2(p_1) - (1-p_1) \log_2(1-p_1)$$
    

#### (2). 로그 밑(Base)을 2로 사용하는 이유

- 밑이 2인 로그($\log_2$)를 사용하여 그래프의 최대 높이(최대 불순도 값)를 깔끔하게 **1**로 맞추기 위함입니다. (자연로그 $\ln$을 써도 동작하지만 최대값이 1로 떨어지지 않아 해석이 다소 까다로워집니다.)
    

#### (3). 예외 처리 연산 ($0 \log_2 0$)

- $p_1 = 0$ 또는 $p_0 = 0$일 때 $\log_2(0)$은 수학적으로 정의되지 않지만, 엔트로피 계산에서는 $0 \log_2 0 = 0$으로 예외 정의하여 처리합니다.
    
---
### 3). 기타 불순도 지표

#### (1). 지니 불순도 (Gini Impurity)

- 엔트로피 외에도 오픈소스 패키지에서는 지니 기준(Gini Criteria)이라는 지표도 자주 사용됩니다.
    
- 지니 기준 역시 엔트로피와 매우 유사한 곡선 모양을 가지며, 실무에서 의사결정나무를 만들 때 훌륭하게 작동합니다.

## 2. Choosing a split : Information Gain

### 1). 정보 이득(Information Gain)의 개념

#### (1). 정의 및 목적

- **정보 이득 (Information Gain):** 특정 특징(Feature)을 기준으로 분할했을 때 **줄어드는 불순도(엔트로피)의 양**을 의미합니다.
    
- **분할 기준 선택:** 여러 특징 중 **정보 이득(엔트로피 감소량)이 가장 큰 특징**을 분할 기준으로 선택하여 데이터의 순도를 가장 높이는 방향으로 트리를 생성합니다.
    

#### (2). 가중 평균 엔트로피 (Weighted Average Entropy)

- 단순히 좌/우 가지의 엔트로피 평균을 내는 것이 아니라, **각 가지에 속한 데이터 개수의 비율**을 반영하여 가중치를 부여합니다.
    
- 데이터 수가 많은 가지의 엔트로피가 높아지면 전체에 미치는 영향이 더 크기 때문입니다.
    
---
### 2). 정보 이득 계산 수식 및 공식

#### (1). 계산 수식

정보 이득은 (분할 전 노드의 엔트로피) - (분할 후 자식 노드들의 가중 평균 엔트로피)로 구합니다.

$$I.G. = H(p_1^{\text{root}}) - \left[ w^{\text{left}} H(p_1^{\text{left}}) + w^{\text{right}} H(p_1^{\text{right}}) \right]$$

#### (2). 주요 변수 정의

- **$p_1^{\text{root}}$:** 분할 전(루트 노드) 양성 데이터 비율
    
- **$w^{\text{left}}, w^{\text{right}}$:** 전체 데이터 중 좌/우 자식 노드로 이동한 데이터의 비율
    
- **$p_1^{\text{left}}, p_1^{\text{right}}$:** 좌/우 자식 노드 내에서의 양성 데이터 비율
    
---
### 3). 정보 이득을 활용하는 이유 및 의의

#### (1). 분할 정지 조건 (Stopping Criteria)

- 정보 이득을 계산하면 단순히 자식 노드의 엔트로피만 확인하는 것과 달리 ‘분할을 통해 불순도가 얼마나 개선되었는가’를 정량 측정할 수 있습니다.
    
- 만약 정보 이득이 특정 기준치(Threshold)보다 너무 작다면, 분할을 멈추어 불필요한 트리의 성장을 막고 **오버피팅(Overfitting)을 방지**합니다.

## 3. Putting it together

### 1). 의사결정나무 전체 학습 절차

#### (1). 전체 구축 프로세스

- **루트 노드 생성:** 전체 훈련 데이터를 루트 노드에 두고 시작합니다.
    
- **최적 특징 선택:** 가능한 모든 특징(Feature)에 대해 정보 이득(Information Gain)을 계산한 뒤, 가장 높은 정보 이득을 제공하는 특징을 선택합니다.
    
- **데이터 분할 및 가지 생성:** 선택한 특징의 값에 따라 데이터를 하위 그룹으로 나누고, 좌/우 가지(Branch)를 생성하여 해당되는 분할 데이터를 보냅니다.
    
- **재귀적 반복 (Recursion):** 정지 조건(Stopping Criteria)을 충족할 때까지 하위 자식 노드에서 위의 분할 과정을 계속 반복합니다.
    

#### (2). 컴퓨터 과학에서의 재귀적 성격 (Recursive Algorithm)

- 전체 트리를 만드는 과정은 하위의 **작은 의사결정나무(Sub-trees)들을 만들어 조합하는 재귀적 과정**입니다.
    
- 실제 알고리즘을 직접 구현할 때 자기 자신을 호출하는 재귀 함수(Recursive Function) 구조가 주로 사용됩니다.
    

### 2). 분할 정지 조건 (Stopping Criteria)

다음 조건 중 하나 이상이 만족되면 분할을 멈추고 최종 예측값을 내놓는 리프 노드(Leaf Node)를 생성합니다.

- **노드의 순도 100% 달성:** 노드 내 모든 데이터가 단 하나의 클래스로만 이루어져 엔트로피가 0이 되었을 때.
    
- **최대 깊이(Maximum Depth) 도달:** 트리의 깊이가 사전에 설정한 임계값에 도달했을 때.
    
- **정보 이득 미흡:** 분할 시 얻을 수 있는 추가적인 정보 이득(Information Gain)이 설정한 기준치 이하일 때.
    
- **샘플 수 부족:** 노드에 남아 있는 데이터(샘플) 개수가 설정한 임계값보다 적을 때.
    

### 3). 트리의 깊이(Depth) 및 하이퍼파라미터 조절

#### (1). 최대 깊이 (Maximum Depth)의 영향

- **깊이가 커질수록:** 트리가 복잡한 패턴을 학습할 수 있게 되지만, 다항식의 차수를 높이거나 신경망을 키우는 것과 같이 **오버피팅(Overfitting) 위험이 증가**합니다.
    
- **하이퍼파라미터 튜닝:** 이론적으로 교차 검증(Cross-validation)을 통해 최선의 `Max Depth`를 선택할 수 있으며, 오픈소스 라이브러리(Scikit-learn 등)에서는 이를 조절할 수 있는 기본 설정값을 제공합니다.

## 4. Using one-hot encoding of categorical features
### 1). 범주형 특징 처리: 원-핫 인코딩 (One-Hot Encoding)

#### (1). 범주가 3개 이상인 특징의 문제점과 해결책

- **기존 상황:** 특징 값이 2개(예: 뾰족함/접힘)일 때는 쉽게 이진 분할을 수행했습니다.
    
- **다중 범주 상황:** 특징이 3개 이상의 값(예: 귀 모양 = 뾰족함, 접힘, 타원형)을 가질 경우, 3개 이상의 하위 가지로 나눌 수도 있습니다.
    
- **원-핫 인코딩 적용:** 범주가 $K$개인 특징을 0 또는 1의 값을 가지는 $K$개의 이진 특징(Binary Features)으로 변환하여 문제를 단순화합니다.
    
    - 예: `귀 모양 = 타원형` $\rightarrow$ `[뾰족함: 0, 접힘: 0, 타원형: 1]`
        
- **원-핫(One-Hot)의 의미:** $K$개의 새로운 이진 특징 중 단 하나의 특징만 값 '1(Hot)'을 갖고 나머지는 모두 '0'을 갖는 구조에서 유래했습니다.
    

#### (2). 원-핫 인코딩의 장점 및 범용성

- **의사결정나무 모델의 재사용:** 데이터를 0과 1로 변환함으로써, 기존의 이진 분할 알고리즘을 수정 없이 그대로 적용할 수 있습니다.
    
- **신경망 및 선형 모델 적용 가능:** 원-핫 인코딩은 의사결정나무뿐만 아니라 **신경망(Neural Networks), 로지스틱 회귀(Logistic Regression)** 등 수치형 입력을 요구하는 다른 머신러닝 모델에도 범주형 데이터를 입력할 수 있게 해주는 필수 기법입니다.

## 5. Continuous Valued Features

### 1). 연속형 변수(Continuous Features) 다루기

기존의 범주형 특성(귀 모양, 얼굴 모양 등)과 달리, **동물의 체중(Weight)처럼 어떤 수치든 될 수 있는 연속형 변수**를 의사결정나무에 활용하는 개념입니다. 체중은 고양이와 개를 구분하는 유용한 특성이 됩니다.

#### 2). 최적의 분할 기준(Threshold) 선택 방법

연속형 변수는 "특정 임계값(Threshold) 이하인가?"라는 조건으로 데이터를 두 그룹으로 분할합니다.

1. **후보 임계값 산출:** 데이터를 특성값 순서대로 정렬한 뒤, 인접한 데이터 사이의 중간값(Midpoint)들을 임계값 후보로 설정합니다. _(예: 데이터 10개 기준 9개 후보)_
    
2. **정보 이득(Information Gain) 계산:** 각 후보 임계값별로 분할했을 때의 정보 이득을 구합니다.
    
    - `Weight <= 8`: 정보 이득 $0.24$
        
    - `Weight <= 9`: 정보 이득 **$0.61$** _(최고치)_
        
    - `Weight <= 13`: 정보 이득 $0.40$
        
3. **최종 분할 수행:** 가장 높은 정보 이득을 주는 임계값(`Weight <= 9`)을 선택하고, 이 값이 다른 특성들보다 뛰어날 경우 해당 조건으로 노드를 분할합니다.

## 6. Regressions Trees

### 1). 회귀 나무(Regression Trees)의 개념

기존의 분류(Classification) 문제(예: 고양이인가 아닌가?)와 달리, **동물의 체중(Weight)과 같은 continuous한 수치(y)를 예측하는 회귀 문제**에 의사결정나무를 적용하는 방법입니다.

- **예측값 결정 방법:** 리프 노드(Leaf Node)에 도달했을 때, 해당 노드에 속한 학습 데이터들의 평균값(Average)을 최종 예측값으로 사용합니다.
    

#### 2). 최적의 분할 기준: 분산 감소(Variance Reduction)

분류 문제에서는 불순도를 줄이기 위해 엔트로피(Entropy) 기반의 정보 이득(Information Gain)을 계산했다면, **회귀 나무에서는 데이터의 수치적 흩어짐을 측정하는 분산(Variance)을 감소시키는 특성을 선택**합니다.

1. **분산(Variance) 측정을 통한 평가:**
    
    - 데이터가 얼마나 넓게 퍼져있는지 측정합니다. (숫자의 폭이 좁을수록 분산이 작음)
        
2. **가중 평균 분산(Weighted Average Variance) 계산:**
    
    - 분할 후 좌우 노드에 속한 데이터 비율($w_{left}, w_{right}$)과 각 노드의 분산을 곱하여 평균 분산을 구합니다.
        
3. **분산 감소량(Variance Reduction) 비교:**
    
    - `(루트 노드의 분산) - (분할 후 가중 평균 분산)`을 계산하여 분산이 얼마나 줄어들었는지 평가합니다.
        
    - **예시 (귀 모양, 얼굴 모양, 수염 유무 비교):**
        
        - **귀 모양(Ear Shape) 분할:** 분산 감소량 **$8.84$** _(최고치)_
            
        - **얼굴 모양(Face Shape) 분할:** 분산 감소량 $0.64$
            
        - **수염 유무(Whiskers) 분할:** 분산 감소량 $6.22$
            
4. **최종 분할 수행:** 가장 큰 분산 감소량을 보이는 특성(`Ear Shape`)을 선택하여 노드를 분할하며, 이를 재귀적으로 반복합니다.
    

### 3). 요약 비교

- **분류 나무 (Classification Tree):** 엔트로피 감소량(**정보 이득, Information Gain**)이 가장 큰 특성을 선택 → **범주 예측**
    
- **회귀 나무 (Regression Tree):** 분산 감소량(Variance Reduction)이 가장 큰 특성을 선택 → **평균값(수치) 예측**

# < Tree Ensembles >

## 1. Using multiple decision trees

-  **단일 의사결정나무의 단점을 보완하는 트리 앙상블(Tree Ensemble)의 개념**에 관한 내용입니다.

### 1). 트리 앙상블(Tree Ensemble)의 필요성

단일 의사결정나무(Single Decision Tree)는 **데이터의 작은 변화에도 매우 민감하게 반응(Sensitive)한다는 약점**이 있습니다.

- **단일 트리의 한계:** 10개의 데이터 중 단 1개의 데이터만 수정되어도 최적의 분할 특성이 완전히 달라질 수 있습니다. (예: `Ear Shape` 기준 분할 → `Whiskers` 기준 분할)
    
- **결과:** 루트 노드가 바뀌면 트리의 전체 구조가 완전히 새로 만들어지므로, 예측의 안정성(Robustness)이 떨어집니다.
    

#### 2). 트리 앙상블의 동작 원리 (다수결 투표)

여러 개의 서로 다른 의사결정나무를 만들고, 이들의 예측 결과를 모아 다수결 투표(Majority Vote)로 최종 결과를 결정합니다.

1. **여러 트리 생성:** 데이터 구조가 조금씩 다른 다양한 트리들을 준비합니다.
    
2. **개별 예측 수행:** 새로운 데이터가 입력되면 각 트리가 독립적으로 분류 결과를 예측합니다.
    
    - **트리 1:** 고양이 (Cat)
        
    - **트리 2:** 고양이 아님 (Not Cat)
        
    - **트리 3:** 고양이 (Cat)
        
3. **다수결 투표 (Majority Voting):** 2:1 결과에 따라 최종 예측을 '고양이(Cat)'로 결정합니다.
    

### 3). 핵심 요약

> **앙상블(Ensemble)의 장점** 개별 트리가 잘못된 예측을 하더라도, 다른 여러 트리의 투표를 통해 오류를 상쇄시킵니다. 이를 통해 모델의 **민감도를 낮추고 예측 정확도와 안정성을 대폭 향상**시킵니다.

## 2. Sampling with replacement

이번 강의는 트리 앙상블을 구축하기 위한 핵심 기법인 복원 추출(Sampling with Replacement)에 관한 내용입니다.

### 1). 복원 추출(Sampling with Replacement)의 개념

주머니에서 공을 뽑은 뒤 **다시 넣고(Replacement)** 다음 공을 뽑는 확률적 샘플링 방식입니다.

- **비복원 추출과의 차이:** 만약 뽑은 데이터를 다시 넣지 않는다면, 전체 데이터 크기만큼 뽑았을 때 결국 원본 데이터와 100% 동일한 집합만 나오게 됩니다.
    
- **특징:** 데이터를 뽑고 다시 넣기 때문에 **특정 데이터가 여러 번 중복 선택**될 수 있고, **일부 데이터는 한 번도 선택되지 않을 수** 있습니다.
    

#### 2). 머신러닝 데이터셋에의 적용 방법

기존 10개의 데이터셋이 있을 때, 복원 추출을 통해 원본과 비슷하면서도 조금씩 다른 새로운 데이터셋을 만듭니다.

1. **동일한 크기의 새 데이터셋 생성:** 원본 데이터 크기가 10개라면, 복원 추출을 10번 반복하여 크기가 10인 새로운 학습 데이터셋을 구성합니다.
    
2. **데이터 중복 및 생략 발생:**
    
    - 2번째 뽑힌 데이터와 5번째 뽑힌 데이터가 완전히 동일할 수 있습니다. (중복 허용)
        
    - 원본 데이터 중 일부는 새 데이터셋에 포함되지 않을 수 있습니다.
        
3. **새로운 데이터셋 완성:** 결과적으로 원본과 **유사하지만 서로 다른 여러 개의 무작위 학습 데이터셋**을 만들어낼 수 있습니다.
    

### 3). 핵심 요약

> **트리 앙상블에서의 역할**
> 
> 복원 추출을 사용하면 원본 데이터셋 하나만으로도 **조금씩 구조가 다른 수많은 무작위 데이터셋**을 만들어낼 수 있으며, 이는 다양하고 튼튼한 트리 앙상블(랜덤 포레스트 등)을 만드는 핵심 기반이 됩니다.

## 3. Random forest algorithm

- 이번 강의는 대표적인 트리 앙상블 알고리즘인 배깅(Bagged Decision Trees)과 랜덤 포레스트(Random Forest)에 관한 내용입니다.

### 1). 배깅(Bagged Decision Trees) 알고리즘

복원 추출(Sampling with replacement)을 활용해 여러 개의 의사결정나무를 만드는 앙상블 기법입니다. _(Bagging = **B**ootstrap **Agg**regat**ing**)_

- **학습 방식:** 원본 데이터 크기 $m$에 대해, 복원 추출로 크기 $m$인 무작위 데이터셋을 $B$번 생성하여 $B$개의 서로 다른 트리를 학습시킵니다.
    
- **트리 개수($B$) 선택:** 보통 $B = 64 \sim 128$개(약 100개 내외)를 권장합니다. $B$가 너무 크면(예: 1,000개) 계산 속도만 느려지고 성능 향상 효과는 미비합니다.
    
- **예측 방식:** 새로운 데이터가 들어오면 $B$개 트리의 다수결 투표(Majority Voting)로 최종 결과를 결정합니다.
    

#### 2). 랜덤 포레스트(Random Forest): 무작위 특성 선택 추가

배깅 알고리즘도 루트 노드 근처에서는 항상 비슷한 특성(Feature)만 선택해 트리들이 비슷해지는 경향이 있습니다. 이를 해결하여 **트리 간의 다양성을 더욱 극대화한 알고리즘이 랜덤 포레스트**입니다.

1. **특성 무작위 선택 (Feature Randomization):**
    
    - 각 노드에서 분할할 특성을 찾을 때, 전체 $n$개 특성을 모두 비교하지 않습니다.
        
    - 무작위로 $k$개의 특성 서브셋($k < n$)을 먼저 뽑은 뒤, 그 $k$개 중에서 가장 정보 이득(Information Gain)이 높은 특성을 선택합니다.
        
2. **권장 $k$값 기준:**
    
    - 전체 특성 수 $n$이 많을 때, 일반적으로 **$k = \sqrt{n}$** 개의 특성을 무작위 추출합니다.
        
3. **강력한 안정성(Robustness):**
    
    - 데이터셋의 변형뿐만 아니라 **사용할 특성까지 무작위성**을 주어, 개별 트리가 서로 크게 달라지게 만듭니다.
        
    - 이에 따라 일부 데이터 변화에 흔들리지 않는 매우 강력하고 정확한 예측 모델이 완성됩니다.
        

### 3). 요약 비교

- **배깅 (Bagged Trees):** 복원 추출로 **데이터만 무작위**로 바꿔가며 여러 트리 생성
    
- **랜덤 포레스트 (Random Forest):** 복원 추출(**데이터 무작위**) + 노드 분할 시 특성도 무작위($k = \sqrt{n}$)로 골라 트리 다양성 극대화

## 4. XGBoost


- 이번 강의는 현재 머신러닝 분야에서 가장 널리 쓰이는 강력한 트리 앙상블 알고리즘인 **부스팅(Boosting)과 XGBoost**에 관한 내용입니다.

### 1). 부스팅(Boosting)의 핵심 개념

부스팅은 이전 트리가 **오답을 낸 "어려운 데이터"에 집중하여 다음 트리를 순차적으로 학습**시키는 기법입니다.

- **의도적 연습 (Deliberate Practice) 비유:** 피아노를 칠 때 전체 곡을 반복하는 것보다 못 치는 마디만 계속 연습하는 것이 효율적이듯, 알고리즘도 **틀린 문제(오분류 데이터)에 더 높은 확률/가중치**를 부여하여 반복 학습합니다.
    
- **배깅(Bagging)과의 차이:** 배깅(랜덤 포레스트)은 트리를 독립적·병렬적으로 만드는 반면, 부스팅은 앞선 트리의 결과를 바탕으로 순차적(Sequential)으로 만들어 나갑니다.
    

#### 2). XGBoost (Extreme Gradient Boosting)의 특징 및 사용법

XGBoost는 부스팅 알고리즘을 매우 빠르고 효율적으로 구현한 오픈소스 라이브러리로, 캐글(Kaggle) 경진대회나 실무에서 딥러닝과 함께 가장 높은 성능을 보이는 모델입니다.

1. **가중치 부여 방식:** 실제 XGBoost는 매번 데이터를 복원 추출하지 않고, 각 학습 데이터에 가중치(Weight)를 다르게 부여하는 방식을 사용해 연산 속도를 대폭 향상시켰습니다.
    
2. **과적합 방지 (Regularization):** 모델 내부에 **규제(Regularization)** 기능이 기본 탑재되어 있어 오버피팅을 효과적으로 방지합니다.
    
3. **코드 사용법 (Python):**
    
    - **분류 (Classification):** `xgboost.XGBClassifier()`
        
    - **회귀 (Regression):** `xgboost.XGBRegressor()`
        
    - 기본 라이브러리를 불러온 뒤 `fit()` 및 `predict()` 메서드로 간편하게 모델을 학습하고 예측할 수 있습니다.
        

### 3). 요약 비교

- **랜덤 포레스트 (Random Forest):** 여러 트리를 **독립적**으로 만들어 **다수결 투표**
    
- **XGBoost (Boosting):** 이전 트리가 틀린 문제에 가중치를 두어 **순차적**으로 오차 개선

## 5. When to use decision trees


### 1). 의사결정나무 vs 신경망(딥러닝) 선택 기준

두 모델 모두 매우 강력하지만, 데이터의 형태(Structure)에 따라 적합한 모델이 다릅니다.

- **정형 데이터(Structured / Tabular Data):**
    
    - 엑셀/스프레드시트 형태의 데이터 (예: 주택 크기, 방 개수, 연식 등)
        
    - 의사결정나무 및 트리 앙상블(XGBoost)이 매우 뛰어나며, **학습 속도가 빠르고 효율적**입니다.
        
- **비정형 데이터(Unstructured Data):**
    
    - 이미지, 비디오, 오디오, 텍스트 등
        
    - 신경망(Neural Network)이 월등한 성능을 발휘합니다.
        

#### 2). 알고리즘별 장단점 및 특징 비교

|**분류**|**의사결정나무 / 트리 앙상블 (XGBoost)**|**신경망 (Neural Network)**|
|---|---|---|
|**적합한 데이터**|**정형 데이터(Tabular Data)**에 최적화|**비정형 데이터** + 정형/혼합 데이터 모두 가능|
|**학습 속도**|**매우 큼 (빠름)** $\rightarrow$ 반복적 모델 개선에 유리|비교적 느림 (대형 모델은 오랜 시간 소요)|
|**해석 가능성**|단일 트리는 시각화 및 직관적 해석 가능|블랙박스 모델로 해석이 어려움|
|**전이 학습**|전이 학습(Transfer Learning) 불가|**전이 학습 가능** (작은 데이터셋에 유용)|
